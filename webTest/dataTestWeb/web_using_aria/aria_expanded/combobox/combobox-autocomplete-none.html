Editable Combobox without Autocomplete Example

The below combobox that enables users to choose a term from a hypothetical list of previously searched terms demonstrates the [a] href="../../#combobox">ARIA design pattern for combobox.[/a] The design pattern describes four types of autocomplete behavior. This example illustrates the autocomplete behavior known as no autocomplete. The terms that appear in the listbox popup are not related to the string that is present in the textbox. In this implementation, the listbox popup is not automatically triggered when the textbox receives focus; the list is opened when the user types a character into the textbox or through an explicit open command.
